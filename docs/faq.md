# ❓FAQ

## 从 0 到入服：Windows 常见问题

本页按“从 0 到成功进入服务器”的顺序，列出 Windows 玩家常见的问题与解决方案。实际操作如与群公告/客户端说明冲突，请 **以服务器公告为准**。

### 1. 我刚装好电脑，想玩服务器，需要准备什么？

- **操作系统**：Windows 10/11 64 位，系统已完成更新。
- **显卡驱动**：推荐从显卡官网（NVIDIA / AMD / Intel）安装最新版驱动，而不是只用 Windows 自动驱动。
- **Java（如果需要）**：
  - 本服目前：第一区服为 **1.20.1**，第二区服为 **1.21.1**，均推荐使用 **64 位 JDK 21** 启动。
  - 如果服务器提供了一键客户端/自带 Java，请优先使用客户端自带 Java。
  - 如不确定，请查看群公告或询问管理。

### 2. 启动器/客户端打不开或被 Windows 提示不安全怎么办？

常见现象：双击无反应、被 Windows SmartScreen 拦截、被杀毒软件直接删掉。

解决方案：

- 从 **群公告指定链接** 或 **官网** 下载客户端，避免来路不明的压缩包。
- 如果出现 SmartScreen 蓝色提示：
  - 点击“**更多信息**” → “**仍要运行**”。
- 如果被杀毒软件误报：
  - 在杀毒软件/Windows Defender 中将客户端所在整个文件夹加入 **信任/排除列表**，然后重新解压并运行。

### 3. 客户端解压或安装失败，路径有中文/空格会有问题吗？

一些启动器或整合包对路径比较敏感，可能在以下情况出问题：

- 安装在 `桌面`、`下载` 等带中文路径的目录下。
- 路径中包含大量空格或非常长。

建议：

- 在磁盘根目录新建一个简单路径，例如：`C:\Minecraft`、`C:\Games\RIME`。
- 将整合包或一键客户端解压到上述目录后再运行。

### 4. 游戏一启动就闪退/回到启动器，没有任何提示？

常见原因：

- 分配内存过少或过多。
- 使用了错误版本的 Java（例如 32 位、Java 8 启动 1.20+ 整合包）。

排查步骤：

1. 在启动器设置里确认：
   - 使用的是 **64 位 Java**（通常路径里带有 `Program Files` 而不是 `Program Files (x86)`）。
   - 分配内存：
     - 最低 **4G**（4096 MB），推荐 **6–8G**，不要分配到总内存的 80% 以上。
2. 如果服务器提供了一键客户端：
   - 优先使用其默认 Java 和默认内存设置。
3. 如果依然闪退：
   - 打开整合包目录下的 `logs/latest.log`，将日志发送给管理或按群内模板反馈。

### 5. 加载游戏很慢 / 卡在某个模组加载界面怎么办？

- 首次启动大型整合包加载时间较长（3–10 分钟属正常）。
- 如果长期卡在同一个模组：
  - 关闭所有后台大流量程序（浏览器下载、网盘、游戏加速器等）。
  - 确保硬盘剩余空间 **至少 10–20G**，推荐安装在 SSD 上。
  - 检查杀毒软件是否在“实时扫描”游戏目录，如有请添加信任。

### 6. 无法添加服务器 / 提示地址无效？

请确认：

- 服务器地址拼写完全正确（可以从 [快速开始](/getting-started) 页面复制粘贴）。
- 不要在地址后面手动加空格、中文字符或多余符号。
- 一般情况下无需手动输入端口，除非公告中特别说明。

如果仍然提示 `Unknown host`：

- 检查本机是否能正常上网，尝试访问常见网站。
- 如使用了代理/加速器，尝试关闭后重启游戏再试。

### 7. 进入服务器时提示版本不一致 / Mod 不匹配？

常见错误信息：`Mod rejections`、`Mismatched Mod Channel List`、`This server has mods that you don't have` 等。

解决方案：

- 确保你使用的是 **服务器指定的整合包/客户端**，不要自己在原版上随便加模组。
- 不要擅自删除或添加整合包内的模组（包括小地图、光影相关模组）。
- 如果服务器更新了版本，请按照公告重新下载最新客户端或更新补丁。

### 8. 可以进服务器，但走路一卡一卡 / 掉帧严重？

客户端卡顿通常有几种情况：

- **画质设置过高**：
  - 适当降低：渲染距离、粒子效果、图像质量、平滑光照等。
- **分配内存不合理**：
  - 过少：会频繁 GC 卡顿；过多：会导致系统整体卡。
- **没有开启独立显卡**：
  - 笔记本用户请在显卡控制面板中将启动器/Java 设为“高性能独立显卡运行”。

如果是“网络卡”而不是“画面卡”：

- 查看右上角延迟（Ping）是否过高。
- 关闭其他占网速的程序或下载。
- 如路由器/线路质量较差，可尝试换网络或使用网络加速服务。

### 9. 一定要正版账号吗？

- 本服使用 **LittleSkin** 作为登录/皮肤服务，你需要在启动器中配置 LittleSkin 账户（或使用服务器提供的一键客户端中的 LittleSkin 登录）。
- 如无特殊说明，仍推荐使用 **正版账号** 并在 LittleSkin 中完成绑定，以避免认证问题和后续兼容性问题。
- 无论是否要求正版，都请不要在公开频道传播盗版启动器下载链接。

### 10. 自己解决不了问题时，反馈需要准备什么信息？

为了方便管理快速定位问题，建议在反馈时附带：

- **简要描述**：你在做什么操作时出错（例如“点启动器后马上闪退” / “加载到 3/7 就卡住不动”）。
- **完整报错截图**，尽量包含窗口上方标题和错误信息。
- **日志文件**：
  - 例如整合包目录下的 `logs/latest.log`，按群公告说明打包或上传到网盘。
- **你的电脑配置**：CPU、内存大小、显卡型号、是否使用机械硬盘等。

准备好以上信息能大幅减少来回询问，让你更快重新回到服务器里玩耍。
